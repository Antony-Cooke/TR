WHAT IS CSS?

  • CSS (Cascading Style Sheets) was created by the World Wide Web Consortium (W3C) to solve the problem of formatting and styling a document.
  • CSS is used to define styles for your web pages, including the design, layout and variations in display for different devices and screen sizes. 
  • This allows HTML to do its job - defining the content of a document.
  • Examples of CSS designs…
    • https://pattle.github.io/simpsons-in-css/
    • http://www.csszengarden.com/

LINKING YOUR CSS AND HTML CODE

  • Put the css inline in the html
  • Put the css in a <style></style> section in the <head>
  • Link to a separate css file in the <head>
  • With an external stylesheet file, you can change the look of an entire website by changing just one file!
  • Demo - linking css file
  • By separating these CSS rules into their own file you:
    1. Reduce repetition in your code
    2. Reduce the amount of information that has to be sent to the browser for each page
      If the CSS file applies to the whole site, it only needs to be sent to the visitor once.

  • Task 1
    • Create a style.css file and create a link to it in your index.html
    • Write some CSS in the file to check it is linked

MORE ABOUT THE HTML <LINK> TAG

  • Absolute links - external links include the complete url to the resource you’re linking to.
    Absolute links start with either http:// or https://.
  • Document-relative links - contain the path to the resource relative to the file where the link is written. Document-relative links don’t begin with /:

<ul id="products_list">

<li id="product_item">Black Forest Gateaux</li>

<li id="product_item">Sticky Toffee Pudding</li>

<li id="product_item">Death-by-Chocolate Cake</li>

</ul>

Both ID and Class are used to add some information to the HTML tags.
The difference is that ID should specify a unique element on the page,
whereas multiple elements can share the same class.
This is useful when you have loads of code to sift through

<DIVS> AND <SPANS>

  • <div> is a block element
  • A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).
  • <span> is an inline element
  • An inline element does not start on a new line and only takes up as much width as necessary.
